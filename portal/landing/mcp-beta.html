Positioning

MCP is the execution layer — not the launch product.

The project earns trust through the local Skill, Docs, Academy and the public Registry. The hosted layer only adds value where teams need shared context, governed tools, auditability and collaboration.

Readiness checklist

What must be true before any production claim.

RequirementBar to clear
PersistencePostgres-backed durable tenant state

Load testsReal pass/fail under realistic concurrency
Governance & access

Least privilege by default.

Governance model
Narrow business actions
MCP tools expose scoped business actions, not universal shell power. Resources are scoped to tenant and workflow; human review stays required for high-impact or irreversible actions.
