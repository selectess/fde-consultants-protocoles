# FDE Consultant Skill — Specification

> **Version**: 1.0.0 (2026-06-22)
> **License**: Apache-2.0
> **Reference**: [agentskills.io](https://agentskills.io) — Anthropic's Agent Skills specification
> **Status**: Living document. Last updated when the FDE methodology evolves.

This document defines the canonical specification for skills in the FDE Consultant ecosystem. It is designed to be **compatible** with the Anthropic Agent Skills standard while extending it with FDE-specific concepts (FDE Assurance Score, DeepSCR, 4-stage loop).

## What is a Skill?

A **Skill** is a portable package that teaches an AI coding agent how to perform a specific task repeatably. Skills contain:

- **SKILL.md** (mandatory): YAML frontmatter + Markdown body
- **scripts/** (optional): Python tools the agent can invoke
- **templates/** (optional): Markdown templates for deliverables
- **references/** (optional): On-demand knowledge files
- **examples/** (optional): Concrete usage examples
- **tests/** (optional): Pytest assertions for verification

## SKILL.md Specification

### YAML frontmatter (mandatory fields)

```yaml
---
name: <kebab-case-skill-name>      # lowercase, hyphens, no spaces
description: <1-sentence description, max 200 chars>
version: <semver, e.g., 1.0.0>
license: <Apache-2.0 | MIT | BSL>
---
```

### Optional fields

```yaml
---
name: my-skill
description: ...
version: 1.0.0
license: Apache-2.0
triggers:                              # keywords that auto-activate the skill
  - "keyword1"
  - "keyword2"
domain: software-engineering           # category
author: <name and email>
mcp_server: <name of MCP server>       # if exposed as MCP tools
trust_score_self: <integer 0-100>       # auto-applied FDE Assurance Score
---
```

### Body structure (recommended)

```markdown
# <Skill Name>

> <1-sentence tagline>

## When to use this skill
<bulleted list of trigger conditions>

## What this skill produces
<1 paragraph>

## Operating Principles
<3-7 principles relevant to this skill>

## Anti-Patterns
<3-7 anti-patterns to avoid>

## Workflow
<numbered steps>

## Examples
<2-3 concrete input/output>

## FDE Assurance Score
<4 components: claim, contradiction, evidence, antipatterns>
```

## FDE-Specific Extensions

### FDE Assurance Score

Every Skill in the FDE Consultant ecosystem must end with a `## FDE Assurance Score` section computed per the DeepSCR protocol:

```
FDE Assurance Score = 25×(claim_present) + 25×(≥3 failure modes) + 30×(evidence trail) + 20×(anti-patterns clean)
```

**Verdicts**:
- **85-100**: Certified — Ship with FDE Assurance Score badge.
- **60-84**: Needs revision — return to Stage 1 of the FDE loop.
- **0-59**: Rejected — falsify claim or abandon.

### 4-Stage FDE Loop

Every Skill should declare which stage(s) of the FDE loop it serves:

1. **Stage 1 — Scoping**: 6-Q decomposition + ROI estimate
2. **Stage 2 — Prototyping**: Held-out promotion gate
3. **Stage 3 — Production**: Production-ready handoff
4. **Stage 4 — Feedback**: FDE Assurance Score certification

### Agent Compatibility

Skills should declare which agents they support via manifests:

- **Claude Code**: `skill/.claude-plugin/plugin.json`
- **OpenAI Codex CLI**: `skill/codex.yaml`
- **OpenClaw / Hermes**: `skill/agents.manifest.json`
- **Cursor / Windsurf** (project-level): generated by `modex/generate.sh`
- **Web LLMs** (ChatGPT, Gemini): copy-paste from `skill/ZERO-INSTALL.md`

## Versioning

- Skills follow [SemVer 2.0](https://semver.org/)
- Breaking changes (incompatible frontmatter) require major version bump
- New tools or templates require minor version bump
- Bug fixes or doc updates require patch version bump

## Security

- Skills MUST NOT contain secrets, API keys, or tokens
- Skills MUST be signed with ed25519 (recommended for paid tiers)
- Skills MUST declare their license in frontmatter
- Skills MUST NOT execute untrusted code without sandboxing

## Compatibility

This spec is **forward-compatible** with Anthropic's [agentskills.io](https://agentskills.io). Any skill compliant with both this spec and agentskills.io will work on:

- Claude Code
- Claude.ai
- Cursor
- Windsurf
- OpenAI Codex CLI
- OpenClaw
- Continue.dev
- Any MCP-compatible agent
