# Historical Note: Arbor Inspiration

This file is kept for compatibility with earlier FDE Consultant references.

The public and canonical method is now:

- [fde-scientific-search.md](fde-scientific-search.md)
- `scripts/scientific_search.py`

Use the term **FDE Scientific Search** in Skill instructions, docs, Academy, and
marketing. Mention Arbor only as historical inspiration for hypothesis-driven
research logic.

## What Carries Forward

- Generate competing solution hypotheses.
- Keep development evidence separate from held-out validation.
- Promote only candidates that pass the held-out gate.
- Preserve rejected hypotheses as reusable lessons.
- Make the process portable across Claude, Codex, Cursor, Windsurf, Hermes, and
  other coding or personal agents.

## What Not To Claim

Do not claim:

- Arbor runtime integration;
- autonomous scientific research;
- real worktree orchestration;
- parallel executors;
- automatic merge to production.

The FDE version is intentionally smaller, more practical, and tied to production
ownership.
